Purolator Batch Tracker 批量查询插件

物流运营批量运单查询工具 — 傻瓜式使用指南

版本 v1.4.7

1 🛠️ 第一步：安装插件（只需操作一次）

  1. 打开 Chrome 浏览器的扩展程序页面：
    在 Chrome 浏览器地址栏输入 chrome://extensions/ 并按回车。
  2. 开启“开发者模式”：
    在扩展程序页面右上角，找到 “开发者模式” (Developer mode) 开关，将其**开启**。
  3. 加载插件文件夹：
    点击左上角的 “加载已解压的扩展程序” (Load unpacked) 按钮，选择我们为您准备的插件根文件夹（文件夹名称通常为 purolator-tracker）。
  4. 固定插件图标：
    点击浏览器右上角拼图形状的“扩展程序”图标，找到 Purolator Batch Tracker，点击旁边的“图钉”按钮，将其固定在工具栏上。

2 🚀 第二步：日常查单操作流程

1. 激活安全令牌（WAF Token）

由于官方网站有防爬虫防火墙，我们在开始批量查单前，需要先让浏览器获得“安全通行证”：

  • 点击浏览器右上角插件图标 📦，选择 Purolator 渠道。
  • 查看状态栏：
    • 🟢 WAF Token 已就绪：可以直接开始输入单号查询。
    • 🔴 WAF Token 已过期 或 未获取：
      1. 点击旁边的 “点击刷新 →”（或“打开官网”），浏览器会自动打开官方查单页面。
      2. 在官方网页中手动输入任意一个运单号并点击查询一次。
      3. 返回插件界面，状态栏会自动刷新并变成绿色的 🟢 WAF Token 已就绪。

2. 输入运单号并启动

  1. 在插件文本框中粘贴您要查询的运单号（支持一行一个，或用逗号、空格隔开，支持多达数百单）。
  2. 点击 “▶ 开始查询” 按钮。
  3. 此时，系统会自动弹出一个专用于自动化的独立浏览器窗口，并在该窗口中自动帮您打字、滚动和递交查询。

3 ⚠️ 第三步：【黄金法则】挂机避坑指南（非常重要！）

在自动查询运行期间，运营同事可以继续在电脑上做别的工作（如写邮件、聊天、浏览网页等），但请务必遵守以下规则：

🚨 黄金规则一：千万不要“最小化”查询小窗口！
  • 错误操作：点击小窗口左上角的黄/绿色减号按钮，将其收缩到屏幕底部的 Dock 栏或任务栏。这会导致操作系统为了省电直接“冻结”小窗口，让插件彻底卡死。
  • 正确做法：随手把它丢在屏幕后面，让您正在操作的主浏览器窗口直接遮挡盖住它。只要小窗口没有被最小化，它在后台就能以 100% 的速度安全运行。
💡 黄金规则二：如何应对弹出来的人机图片验证？
  • 双重提醒系统：如果查询过程中官网弹出了人机验证码，独立小窗口会自动弹到屏幕最前方并闪烁跳动，同时电脑扬声器会播放清脆的中文语音提醒：“注意！检测到人机验证码，请尽快处理。”。
  • 处理办法：同事只需用鼠标**顺手点选一下验证图片**。一旦验证通过，小窗口会再次自动隐匿，插件会**自动恢复查询**，不需要任何人手动去点暂停或继续！

4 📊 第四步：查看与导出结果

  1. 在插件界面点击 “查看查询结果”（或在浏览器中打开结果页）。
  2. 页面中会实时滚动显示每一个单号的最新物流状态、签收时间、当前位置、异常报错等。
  3. 数据展现升级说明：
    • 预计送达 (ETA)：运单属于未妥投状态且官方有预估时间时显示具体日期；如果官方当前没有计算出来（显示为 Not Available），插件也会对应展示 Not Available，与官网保持完全一致！已妥投（Delivered）件则显示 —。
    • 自提地址：当运单处于可自提状态（Available for pickup）时，**为了确保请求行为参数与真人操作100%完全一致，插件会静默等待网页自身加载自提网点卡片时发起的 location 请求**，并在此刻自动拦截、解析出该站点的具体地址（例如 1550 Creditstone RD, Concord, ON L4K 5N1）并填入该字段！
    • 历史记录 (最新 3 条)：同时格式化并倒序展示最新的 3 条历史轨迹，供更深入的详情排查。
  4. 查询全部完成后，点击右上角的 “导出 Excel (CSV)”，即可一键下载完整的物流明细报表。导出的表格同样包含所有独立地址与 ETA 数据，可极大简化运营统计工作。
